# UI Kit — Mapa Estrela Potiguar

The flagship product: a night-sky map of Rio Grande do Norte where every tourism
attraction is a **star**. Google-Maps-style layout — a results sidebar on the left,
a starry map canvas on the right, and a detail panel that takes over the sidebar
when a star is selected. Ships with a day/night theme toggle.

## Run
Open `index.html`. It loads `../../styles.css` and `../../_ds_bundle.js`, then the
screen modules. (The bundle is generated by the compiler — preview it from the
Design System, not by opening the raw file before a build.)

## Interactions
- **Search** filters by name, city or category.
- **Filter chips** (Praias / Históricos / Dunas / Natureza / Gastronomia) toggle categories.
- **Click a star** (map pin) or a card → opens the **detail panel** (hero, rating, tabs, rota).
- **Theme toggle** (top-right of the map) flips the whole app between night and day maps.
- **Map controls** (bottom-right): zoom, layers, locate.

## Files
- `index.html` — mounts the app, loads bundle + modules.
- `data.js` — sample attractions + categories (`window.EP_ATTRACTIONS`, `window.EP_CATEGORIES`).
- `icons.jsx` — Lucide-style inline icon set (`window.EPIcons`).
- `MapCanvas.jsx` — the vector night map (land, ocean, roads) with star pins.
- `Sidebar.jsx` — brand header, search, filters, results list.
- `DetailPanel.jsx` — selected-attraction detail (tabs, info, reviews, actions).
- `Controls.jsx` — `FilterChips` + floating `MapControls`.
- `MapApp.jsx` — top-level state (theme, query, filters, selection).

## Composition
Built entirely from the design-system primitives: `MapPin`, `AttractionCard`,
`Button`, `IconButton`, `Input`, `Tag`, `Badge`, `Rating`, `Tabs`, `Avatar`,
`ThemeToggle`. No component logic is re-implemented here — only composed.
